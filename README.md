# nous3508.github.io - Jekyll + FlexSearch template

Quick deploy steps:
1. Create repository `nous3508.github.io` (if not exists).
2. Upload these files preserving paths:
   - _config.yml
   - _data/profile.yml
   - _includes/head.html
   - _layouts/default.html
   - index.md
   - assets/css/style.css
   - assets/js/site.js
   - assets/js/search.js
   - _posts/ (your posts)
   - _projects/ (your project pages)
3. Upload your avatar to the repository at `/web-app-manifest-192x192.png` (you said it's at Nous3508/Nous3508.github.io/web-app-manifest-192x192.png — ensure the file is in repo root or adjust the path in index.md).
4. Upload logo, social-card and resume:
   - /assets/images/favicon.png (logo / favicon)
   - /assets/images/social-card.png (OG image)
   - /assets/resume.pdf
5. In GitHub repository Settings → Pages enable Pages for branch main (root).
6. Wait a few minutes and visit https://nous3508.github.io

Placeholders to replace later:
- Analytics IDs in _includes/head.html (GA4 / Plausible / Fathom).
- Social card and favicon files.
- Optional: If you prefer avatar file under assets/images/avatar.png, update index.md accordingly.

Notes:
- FlexSearch client index is built from /search.json (generated by _layouts/search.json). Make sure you keep that layout file if you want site posts/projects indexed.
- Public GitHub API limits apply; for reliable dynamic data you can later add a GitHub token as a build-time secret (advanced).

## 🎨 首页背景粒子效果 — 调节指南

首页使用了 Canvas 粒子动画作为背景动态效果（仅首页出现）。所有参数都在 `assets/js/particles.js` 文件顶部的 `CONFIG` 对象中，打开文件找到下面这段：

```js
const CONFIG = {
  count: 60,            // 粒子数量
  minSize: 1.5,         // 最小尺寸 (px)
  maxSize: 4.5,         // 最大尺寸 (px)
  minSpeed: 0.15,       // 最小飘浮速度
  maxSpeed: 0.45,       // 最大飘浮速度
  minOpacity: 0.15,     // 最小透明度
  maxOpacity: 0.55,     // 最大透明度
  connectionDist: 130,  // 连线距离 (px)，设为 0 则不连线
  colors: [             // 粒子的颜色列表
    { r: 155, g: 92, b: 255 },   // 紫色 #a876f9
    { r: 35,  g: 213, b: 255 },  // 青色 #23d5ff
    { r: 142, g: 92, b: 255 },   // 深紫 #8e5cff
    { r: 100, g: 160, b: 255 },  // 中间蓝紫
  ],
};
```

### 各参数详解

#### 1️⃣ 粒子数量 — `count`
- **默认值：** `60`
- **作用：** 画面中同时存在的粒子个数。
- **调大**（如 `100`）：粒子更密集，星空感更强，但性能开销增加。
- **调小**（如 `30`）：更稀疏简约，适合低性能设备。
- 💡 建议范围：`20` ~ `120`

#### 2️⃣ 粒子大小 — `minSize` / `maxSize`
- **默认值：** `minSize: 1.5`，`maxSize: 4.5`
- **作用：** 每个粒子会在这个范围内随机取一个大小（单位像素）。
- 想让粒子更细碎？把 `maxSize` 降到 `3`。
- 想要明显的星星？把 `maxSize` 提到 `6` 或 `8`。
- 💡 注意：较大的粒子会带发光光晕效果。

#### 3️⃣ 飘浮速度 — `minSpeed` / `maxSpeed`
- **默认值：** `minSpeed: 0.15`，`maxSpeed: 0.45`
- **作用：** 粒子从底部向上飘浮的速度范围。值越大飘得越快。
- **调大**（如 `0.5` ~ `1.0`）：粒子流动感更强，像星空在流动。
- **调小**（如 `0.05` ~ `0.2`）：几乎静止，只有极缓慢的漂移。
- 💡 建议范围：`0.05` ~ `1.0`

#### 4️⃣ 透明度 — `minOpacity` / `maxOpacity`
- **默认值：** `minOpacity: 0.15`，`maxOpacity: 0.55`
- **作用：** 粒子的不透明度范围。值越大粒子越明显。
- **调大**（如 `0.3` ~ `0.8`）：粒子更亮眼，视觉冲击强。
- **调小**（如 `0.05` ~ `0.3`）：粒子若有若无，适合极简风格。
- 💡 注意：透明度还会叠加上粒子自身的闪烁动画，所以最终透明度是动态变化的。

#### 5️⃣ 连线距离 — `connectionDist`
- **默认值：** `130`
- **作用：** 当两个粒子之间的距离小于此值时，它们之间会画一条细线，形成网络效果。单位像素。
- **调大**（如 `200`）：连线更多，网络感更强。
- **调小**（如 `50`）：只有很近的粒子才会连线。
- **设为 `0` 可完全关闭连线**，只保留独立粒子。

#### 6️⃣ 粒子颜色 — `colors`
- **默认值：** 紫色 → 青色渐变列表
- **作用：** 每个粒子会从这个颜色列表中随机选一个作为基础色，并叠加 ±30 的随机偏移，让颜色有自然变化。
- 想换成单色？只保留一项即可，例如：
  ```js
  colors: [
    { r: 255, g: 255, b: 255 },  // 纯白
  ],
  ```
- 想增加颜色种类？添加更多 RGB 对象：
  ```js
  colors: [
    { r: 155, g: 92, b: 255 },   // 紫
    { r: 35,  g: 213, b: 255 },  // 青
    { r: 255, g: 100, b: 100 },  // 粉红
  ],
  ```

### 修改后如何生效？

1. 修改 `assets/js/particles.js` 中的 `CONFIG` 参数。
2. 保存文件。
3. 刷新首页即可看到效果 —— **无需重新构建 Jekyll 站点**，因为这是浏览器端运行的 JavaScript。

### 想完全移除粒子效果？

有两种方式：

**方法 A — 移除脚本加载（推荐）：**
打开 `_layouts/default.html`，删除或注释掉这一行：
```html
<script src="/assets/js/particles.js"></script>
```

**方法 B — 保留代码但禁用：**
在 `assets/js/particles.js` 顶部将 `count` 设为 `0`：
```js
count: 0,
```
这样粒子数量为零，不会有任何绘制，但脚本仍会被加载（多一次网络请求）。
